👋 Welcome! Before You Begin

This tutorial is designed for absolute beginners to MuleSoft. Each example walks you through building a real API or integration in Anypoint Studio (the visual IDE for MuleSoft). You'll learn the core building blocks: HTTP Listener, Set Payload, Database Connector, DataWeave, Error Handling, and more.

🛠️ Prerequisites: Download & install Anypoint Studio from MuleSoft's official site. You'll also need Java 8 or 11 and a MuleSoft account (free trial available).
💡 Tip: Each example below is a separate Mule project. In Anypoint Studio, go to File → New → Mule Project and give it a unique name (e.g., example-01-hello-world).

1 Hello World API – Your First HTTP Endpoint Beginner

Goal: Create a simple API that responds with "Hello, MuleSoft!" when you visit http://localhost:8081/hello.

🧱 Components Used

  • HTTP Listener – Listens for incoming HTTP requests
  • Set Payload – Sets the response body

📝 Step-by-Step

  1. In Anypoint Studio, create a new Mule Project named hello-world-api.
  2. From the Mule Palette (right side), search for HTTP and drag Listener onto the canvas.
  3. Click the HTTP Listener → Click the green + button next to "Connector configuration" → Set Host: 0.0.0.0 and Port: 8081. Click OK.
  4. Set the Path to /hello in the HTTP Listener properties.
  5. From the Palette, drag a Set Payload component after the HTTP Listener.
  6. In Set Payload's Value field, enter: Hello, MuleSoft!
  7. Save and Run the project (right-click → Run As → Mule Application).
  8. Open your browser and navigate to http://localhost:8081/hello — you should see the greeting!

📄 XML Configuration

<flow name="hello-world-flow">
                <http:listener config-ref="HTTP_Listener_config" path="/hello"/>
                <set-payload value="Hello, MuleSoft!"/>
            </flow>
💡 Key Takeaway: Every Mule flow starts with a source (like HTTP Listener) that triggers the flow, followed by processors that handle the logic.

2 REST API with Path Parameters Beginner

Goal: Create an endpoint that greets a user by name: /greet/John returns "Hello, John!".

🧱 Components Used

  • HTTP Listener (with URI parameters)
  • Set Payload (using DataWeave expression)

📝 Step-by-Step

  1. Create a new Mule Project: path-param-api.
  2. Add an HTTP Listener (Host: 0.0.0.0, Port: 8081).
  3. Set the Path to: /greet/{name} — the curly braces {name} define a path parameter.
  4. Add a Set Payload after the listener.
  5. Click the fx button next to Value and enter the DataWeave expression:
    #["Hello, " ++ attributes.uriParams.name ++ "!"]
  6. Run and test: http://localhost:8081/greet/John → "Hello, John!"
